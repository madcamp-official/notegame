# 🎮 게임 서버 실행 가이드 (팀원용)

이 문서 하나만 따라 하면 게임을 돌릴 수 있어요.
게임을 실행하려면 **게임 서버**가 켜져 있어야 하고, 그 서버는 **원격 AI(vLLM)** 에 연결돼야 합니다.
이 가이드는 그걸 자동으로 해주는 스크립트를 쓰는 법입니다.

> **전제:** KAIST VPN 에 이미 접속돼 있어야 합니다. (VPN 없으면 원격 AI 에 못 붙어요.)

---

## 0. 딱 3개만 준비하세요 (처음 한 번)

### ① Node.js 20 이상 설치
- https://nodejs.org 에서 LTS 버전 설치
- 확인:
  ```
  node -v
  ```
  `v20.x` 이상이면 OK.

### ② SSH 개인키 `qwen_key` 배치
팀장에게 **`qwen_key`** 파일을 받아서 아래 위치에 둡니다.

- **Mac / Linux:** `~/.ssh/qwen_key`
  ```bash
  mkdir -p ~/.ssh
  cp <받은_qwen_key_경로> ~/.ssh/qwen_key
  chmod 600 ~/.ssh/qwen_key
  ```
- **Windows:** `C:\Users\<내이름>\.ssh\qwen_key`
  파일을 그 폴더에 복사한 뒤, 권한을 잠급니다(안 하면 SSH가 키를 거부해요):
  ```powershell
  icacls "$env:USERPROFILE\.ssh\qwen_key" /inheritance:r /grant:r "$($env:USERNAME):R"
  ```

### ③ AI 인증키(`VLLM_API_KEY`) 넣기
팀장에게 **`VLLM_API_KEY`** 값(긴 문자열)을 받아둡니다.
아래 2번에서 서버를 처음 실행하면 `.env` 파일이 자동으로 생기는데, 거기에 이 값을 붙여넣으면 됩니다.

---

## 1. 실행하기

터미널을 열고 이 프로젝트의 `Server` 폴더로 이동한 뒤:

### Mac / Linux
```bash
cd Server
./run.sh
```

### Windows
`Server` 폴더에서 **`run.bat` 파일을 더블클릭**
(또는 PowerShell에서)
```powershell
cd Server
powershell -ExecutionPolicy Bypass -File run.ps1
```

### 처음 실행하면?
1. 스크립트가 `.env` 파일을 자동으로 만들어주고 **한 번 멈춥니다.**
2. `Server/.env` 파일을 열어서 아래 줄을 찾아
   ```
   VLLM_API_KEY=
   ```
   `=` 뒤에 팀장에게 받은 값을 붙여넣고 저장합니다.
   ```
   VLLM_API_KEY=여기에_받은_긴_문자열
   ```
3. 다시 한 번 실행(`./run.sh` 또는 `run.bat`)하면 끝.

정상적으로 뜨면 마지막에 이런 메시지가 나옵니다:
```
▶ 서버 시작 (http://127.0.0.1:8787).  Ctrl+C 로 종료.
```
이 터미널 창은 **켜둔 채로** 두세요. (서버가 실행 중인 창입니다.)

---

## 2. Unity(게임) 실행

서버 창을 켜둔 상태에서, Unity 에디터로 프로젝트를 열고 **Play ▶ 버튼**을 누르면
게임이 로컬 서버(127.0.0.1:8787)에 자동으로 연결됩니다.

---

## 3. 코드가 업데이트되면? (중요)

`main` 브랜치에는 게임 플레이 관련 코드가 계속 올라옵니다.
**업데이트를 받는 방법은 매번 똑같습니다:**

```bash
git pull
```

그다음 평소처럼 다시 실행하면 됩니다.

```bash
cd Server
./run.sh      # Windows 는 run.bat
```

> 실행 스크립트가 매번 `npm install` 을 자동으로 돌려서
> **새로 추가된 패키지도 알아서 반영**합니다.
> 즉, 이 가이드는 앞으로 코드가 바뀌어도 계속 그대로 유효합니다.

---

## 4. 잘 안 될 때

| 증상 | 원인 / 해결 |
|---|---|
| `SSH 키가 없습니다` | `qwen_key` 를 위치(0-②)에 안 뒀거나 경로가 다름 |
| SSH가 키를 거부(`bad permissions`) | Mac: `chmod 600 ~/.ssh/qwen_key` / Windows: 위 `icacls` 명령 실행 |
| `VLLM_API_KEY 가 비어 있습니다` | `Server/.env` 에 값 안 넣음 (0-③, 1번 참고) |
| 터널은 열렸는데 AI 응답이 없음 | **VPN 이 꺼져 있는지** 먼저 확인 |
| `node: command not found` | Node.js 미설치 (0-①) |
| Windows에서 `run.ps1 실행 차단` | `run.bat` 으로 실행하거나 `-ExecutionPolicy Bypass` 사용 |
| 포트 8787 이미 사용 중 | 이전에 켜둔 서버 창이 있는지 확인, 있으면 그거 쓰면 됨 |

### 터널만 따로 끄고 싶을 때
- **Mac/Linux:** `pkill -f '8000:127.0.0.1:8000'`
- **Windows:** 작업 관리자에서 `ssh.exe` 종료
